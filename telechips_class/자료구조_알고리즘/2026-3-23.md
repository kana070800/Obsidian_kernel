이진트리
---
우리가 주로 볼 트리구조
하나의 노드가 자식 노드를 2개까지만 가지는 트리 > degree =2

종류
	포화 이진트리 > 잎노드들이 모두 같은 깊이
	완전 이진트리 > 잎노들들이 트리 왼쪽에서부터 채워짐
	높이 균형 트리 > root를 기준으로 양쪽 하위 트리의 높이가 2 이상 차이 나지 않는 트리
		불균형 트리 > 차이가 2 이상 발생, 탐색속도의 불균형
	완전 높이 균형 트리 > root 기준 양쪽 하위 트리 높이가 동일



저장 시 정렬 기준이 일관성 있게 정해진다면 탐색이 수월해진다
	기존 너비우선이나 깊이우선처럼 일일이 확인 안해도 됨
	데이터 업데이트, 추가가 주기적으로 이뤄질 시 > 한쪽의 깊이만 증가
		주기적인 재정렬이 필요
			ex ) 레드 블랙 트리

트리 출력 시 차이점 분석 
```
void Tree_Porder_Print(Tnode* node){
	printf("%c", node->data);

	if ((node->left == NULL)&&(node->right == NULL))
		return;
	Tree_Porder_Print(node->left);
	Tree_Porder_Print(node->right);

	return;
}
//--------------------------------------------------------------
void Tree_Porder_Print(Tnode* node){
	if (!node)
		return;
	printf("%c", node->data);
	Tree_Porder_Print(node->left);
	Tree_Porder_Print(node->right);

	return;
}
//---------------------------------------------------------------
void Tree_Porder_Print(Tnode* node){

	printf("%c", node->data);
	if (node->left)
		Tree_Porder_Print(node->left);
	if (node->right)
		Tree_Porder_Print(node->right);

	return;
}
```

첫번째 경우: 현재 노드의 양쪽 자식이 모두 NULL인 경우만 실행하지 않고 종료함. 하지만 자식이 하나라도 존재할 경우 모든 자식을 재귀호출한다는 단점이 있음

세번째 경우: 왼쪽자식, 오른쪽 자식 "따로" NULL인지 비교하여 NULL이 아닌 경우에만 재귀함수를 호출한다는 장점이 있음. 따라서 첫번째 경우보다 실행 횟수가 더 적어 대규모 데이터인 경우 스택 오버플로우 위험이 더 낮다는 장점이 있다.

두번째 경우: 현재 노드가 NULL인 경우에만 종료하고 아니면 양쪽 자식을 모두 재귀호출한다. 하지만 자식 노드를 모두 호출하기 때문에 실행 횟수 측면에서는 최선이 아님.

첫번째와 세번째 코드의 문제점은 만약 main 함수에서 함수를 호출할 때 처음에 root가 아닌 NULL을 넘겨준다면 처음에 조건 검사가 없어 printf에서 NULL->data를 출력하려고 해 에러가 발생할 수 있음. 하지만 두번째는 처음부터 NULL 검사가 있기 때문에 안전함.

배열트리
---
root의 인덱스 = 1 일 때 > 연산 수 적다,
	자식의 인덱스
		`left = 2*n
		`right = 2*n + 1`
	부모의 인덱스
		`parent = n/2
root의 인덱스 = 0 일 때
	자식의 인덱스
		`left = 2*n + 1
		`right = 2*n + 2`
	부모의 인덱스
		`parent = (n - 1)/2`

힙
---
힙 순서 속성(Heap Order Property)을 만족하는 완전 이진 트리
	트리 내의 모든 노드가 부모 노드보다 커야 한다는 규칙
힙 내에서 어떤 노드를 찾으려면 모든 노드를 순회해야 하므로 이진 탐색 불가능
힙에 사용되는 연산 
	노드를 삽입하는 연산 
	뿌리 노드를 없애는 ‘최솟값 삭제’ 연산
